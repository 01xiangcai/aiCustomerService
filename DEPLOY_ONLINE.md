# AI 客服中台 · 简约部署指南 (宝塔面板篇)

针对宝塔面板环境，我们推荐以下操作流程。

---

## 0. 环境准备 (宝塔面板)

1.  **安装 Docker**: 进入宝塔【软件商店】-> 搜索“Docker” -> 安装【Docker管理器】。
2.  **安装 Java**: 【软件商店】安装【Java项目管理器】或在最新版宝塔【项目】中直接添加 Java 项目。
3.  **安装 Nginx**: 确保已安装 Nginx（用于部署前端和反向代理）。

---

## 1. 基础设施部署 (使用宝塔 Docker)

1.  将项目根目录下的 `docker-compose.yml` 和 `sql/` 文件夹上传到服务器（例如 `/www/wwwroot/ai-cs/`）。
2.  进入宝塔【Docker】菜单 -> 【Compose】选项卡 -> 【添加Compose项目】。
3.  选择刚才上传的 `.yml` 文件，点击“添加”即可自动启动所有数据库和会话环境。

---

## 2. 后端打包与部署 (Java)

1.  **本地打包**: 在项目根目录执行 `mvn clean package -DskipTests`。
2.  **上传文件**: 将 `ai-cs-server/target/ai-cs-server.jar` 上传到服务器。
3.  **创建项目**: 宝塔【网站】-> 【Java项目】 -> 【添加Java项目】。
4.  **环境切换 (prod)**:
    - 在宝塔添加项目界面，找到 **项目执行命令** 或 **参数** 框。
    - 务必填入：`--spring.profiles.active=prod`。
    - 这样程序会读取 `application-prod.yml` 中的配置（如生产数据库、Redis 等）。
5.  **配置核对**: 启动前请确保服务器上的 `application-prod.yml` 已填入正确的 Docker 数据库密码。

---

## 3. 管理后台部署 (宝塔版)

1.  **本地打包**: 在本地 `ai-cs-admin` 目录下运行 `npm run build` 生成 `dist` 文件夹。
2.  **创建站点**: 宝塔【网站】-> 【添加站点】（创建纯静态站点）。
3.  **上传代码**: 将 `dist` 目录内的所有文件上传到该站点的根目录。
4.  **配置反向代理**:
    - 点击该站点的【设置】 -> 【反向代理】 -> 【添加反向代理】。
    - **代理名称**: `api`
    - **目标URL**: `http://127.0.0.1:9900`
    - 代理目录设为 `/api/` (根据前端配置调整)。

> [!NOTE]
> 由于服务器 3306 端口占用，本项目 MySQL 容器映射端口已调整为 **3307**。

---

## 4. 客服组件打包 (Widget)

`ai-cs-widget` 目录下包含 JS 和 UniApp 组件：
- **JS 版**: 直接在网页中引用 `ai-cs-widget.js`。
- **UniApp 版**: 将 `uniapp/` 目录下的组件拷贝到你的 UniApp 项目中使用即可。

---

## 5. 校验与维护

- 查看后端日志：`tail -f server.log`
- 查看容器状态：`docker ps`
